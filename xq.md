#### **1. 功能需求**

系统主要用户分为**管理员**和**读者**两大角色，其功能需求如下：

##### **1.1 管理员 (Administrator) 功能列表**

| 功能模块     | 具体功能点   | 详细描述                                    |
| -------- | ------- | --------------------------------------- |
| **用户管理** |         |                                         |
|          | 冻结/删除   | 冻结读者，或删除不再使用的读者账户。                      |
|          | 查询读者    | 根据学号/工号、姓名等条件查询读者信息及其当前的借阅状态。           |
| **图书管理** | 新增图书    | 录入新采购图书的详细信息，包括书名、作者、出版社、ISBN、馆藏位置、数量等。 |
|          | 修改/删除图书 | 更新图书信息（如馆藏位置变更），或移除已报废、丢失的图书。           |
|          | 查询图书    | 根据书名、作者、ISBN等多种条件快速检索图书信息。              |
| **借阅管理** | 办理借阅    | 为读者办理图书借阅手续，系统自动记录借阅信息并更新图书库存。          |
|          | 办理归还    | 为读者办理图书归还手续，系统自动更新借阅记录和图书库存，并检查是否超期。    |
|          | 借阅记录查询  | 查询系统中所有的借阅历史和当前借阅状态，可按读者或图书进行筛选。        |
|          | 逾期处理    | 系统能自动识别并标记逾期未还的记录，管理员可根据此信息进行催还或罚款处理。   |
| **系统设置** | 设置借阅规则  | 设置不同读者类型的可借数量、借阅期限、可续借次数等。              |

##### **1.2 读者 (Reader) 功能列表**

| 功能模块     | 具体功能点    | 详细描述                              |
| -------- | -------- | --------------------------------- |
| **用户认证** | 登录/登出/注册 | 读者使用手机号和密码登录系统，访问个人功能。            |
|          | 修改密码     | 读者可以修改自己的登录密码，保障账户安全。             |
| **图书查询** | 关键字查询    | 根据书名、作者、出版社、ISBN等关键字搜索馆藏图书。       |
|          | 查看图书详情   | 查看特定图书的详细信息，包括简介、馆藏位置、当前可借阅状态和数量。 |
| **个人中心** | 查看个人信息   | 查看自己的账户信息，如姓名、联系方式等。              |
|          | 查看借阅历史   | 查看自己当前已借阅的图书列表、应还日期以及历史借阅记录。      |
|          | 续借图书     | 在图书到期前，对符合续借规则的图书进行在线续借操作。        |

---

#### **2. 非功能需求**

| 需求类别                   | 详细描述                                                                                                                                            |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| **性能需求 (Performance)** | **响应时间**：核心功能（如图书查询、借阅处理）的服务器响应时间应在2秒以内。**并发处理**：系统应能支持至少100名用户同时在线进行查询和借阅操作，并保持稳定的性能。                                                          |
| **安全性需求 (Security)**   | **身份认证**：所有用户必须凭有效的用户名和密码才能登录系统，密码在数据库中应加密存储。**权限控制**：系统必须实现严格的角色权限分离，读者用户不能访问任何管理员功能（如图书管理、用户管理）。**数据防篡改**：防止未经授权的数据库访问和恶意的数据篡改，如防止读者自行修改借阅记录。 |
| **可用性需求 (Usability)**  | **界面友好**：系统界面应简洁、直观，功能布局合理，无需复杂培训即可上手操作。**操作反馈**：用户执行关键操作（如借阅、归还、续借）后，系统应提供明确的成功或失败提示信息。**响应式设计**：读者端界面应能良好地适应PC、平板和手机等不同尺寸的屏幕，方便随时随地查询。        |
